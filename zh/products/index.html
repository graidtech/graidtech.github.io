# GRAID SupremeRAID™ - 為未來設計的 RAID 卡

# 軟體可組合基礎架構

在組合式的基礎架構(Composable Infrastructure)中，計算、存儲和網絡資源是從其物理位置抽像出來的，並透過基於網頁的界面由軟體進行管理，使數據中心資源像雲端服務一樣容易獲得，並成為私有和混合雲解決方案的基礎。

隨著 NVMe SSD 和 NVMEoF 的出現，我們看到SCI(Software Composable Infrastructure)在不犧牲性能和延遲的情況下將計算與存儲分離的可能性。但是，隨著 NVMe SSD 技術的發展，我們也看到 RAID 數據保護已成為整體系統中的性能瓶頸。

# 效能瓶頸

在過去，人們只能仰賴硬體 RAID 卡或軟體 RAID 來保護資料。在硬碟的時代，RAID 卡上的處理器將足以處理所有I/O。畢竟，即使是使用 SAS 硬碟，性能也只能達到 200 IOPS 左右和 150MB/s 的吞吐量。但是，在 SSD 時代，單個 NVMe SSD 可以為您提供約 100萬 IOPS 和 7GB/s 的吞吐量。這使得硬體 RAID 卡或軟體 RAID 無法趕上 SSD 的性能，最終成為存儲結構最大的瓶頸。

同時，傳統的 RAID 卡需要硬碟或 SSD 通過電纜直接連接到 RAID 卡上。這限制了它在現今軟體組成的基礎架構中的發展。

# 支援組合式的基礎架構

有了 SupremeRAID™ ，將遠端的 SSD 透過 NVMeoF 接入並組合成具有資料保護的卷給應用軟體使用將成為可能

# 將 CPU 留給需要的應用

卸載所有 RAID 計算到 SupremeRAID™ 上，將珍貴的 CPU 資源留給需要大量計算的應用，以提高生產力並節省成本

# 隨插即用

不同於傳統的硬體 RAID 卡， SupremeRAID™ 能在無須更改伺服器硬體佈局的情況下保護任何型態的 NVMe SSD，可減少許多硬體客製化所衍生的潛在風險

# 支援多種的 NVMe 儲存裝置格式

SupremeRAID™ 可支援所有型態的 NVMe 儲存裝置格式，如 U.2 、 M.2 甚至是 AIC，這使 SupremeRAID™ 成為世界上兼容性最高的 NVMe SSD RAID 卡

# 易於維護

由於傳統的硬體 RAID 卡所搭載的處理器效能非常得有限，因此會需要搭載記憶體快取來達到宣稱的效能，但為了保持數據完整性，需要使用電池來防止數據在斷電時丟失。如果未搭載電池或電池損壞，RAID 卡將不得不切換成直接寫入的模式，使其效能嚴重下降

SupremeRAID™ 利用高效能的運算處理器與卓越的軟體架構，在無須使用快取記憶體的情況下，可以達到傳統的硬體 RAID 卡 2 到 10 倍的效能，減輕管理記憶體電池的負擔並保持穩定的性能

# 高擴充性

深憶科技特有的軟硬體整合技術，使 SupremeRAID™ 在未來可以透過軟體升級，輕易疊加更多如壓縮（Compress）、加密（Encryption）與空間精簡分配（ Thin Provision）等功能
